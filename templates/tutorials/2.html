Data Types


While technically every piece of information that we supply to computers will inevitably be interpretted as ones and zeroes, it should come to no one's surprise that we still have many ways of modelling data. Java needs to know what data you are throwing around in order to properly handle it in certain scenarios. Like for example, adding two words together is a completely different operation than adding two numbers together.

The chart above shows all 8 primitive types in their full glory–the boolean which can only be one of two values: true or false, the char which simultaneously stores a positive integer value AND its ASCII representation (basically allowing you to do some serious fiddling like adding a number to a character), various implementation of integer-type numbers (the most commonly used one among them would be the int), and floating point numbers (float and double) which are essentially numbers stored in scientific notation.

String is not a primitive type. It is a sequence of characters–by building up from a primitive type it sort of outs itself as a class data type. You can also tell this by two other observations: String types are capitalized, and Strings have callable methods. You will find out more as we learn about Strings in detail.

In this lesson, you will be focusing mainly on the primitive types–the building blocks of any programming language.

Section 1. Variable Declaration

Expand

Before you use data types of any sort, it's first important that you understand that Java has many ways to work with data. For example:

System.out.println(2 + 3);

This piece of code is adding two integers together. The output would be:

5

As you should expect.

However, often we don’t want to just add 2 and 3 specifically–what if we wanted to add two unknown values?

For this, we use variable declarations. Here would be how we replicate the above:

int a = 2;
int b = 3;
System.out.println(a + b);

The output of this piece of code is also 5. But now, we have two variables–a and b–that we can assign values to, and perform mathematical operations with.

There's a lot to digest here, so it's worth it to look at each key word.

Variable declaration goes like this:
<data type> <variable name> = <value>;

Some more examples…

int first = 1;
int second = 2;
    
boolean iDidMyHomework = true;
boolean visitedZoo= false;
    
double x = 3.14;
double y = 2.718;
double z = x-y;
    
char middleInitial = 'R';

Of course, you can call System.out.println() on any of these variables, and perform specific operations on them. 

int first;
first = 1;
    
System.out.println(first);

This is an example of declaring a variable but not initializing it. You can always declare variables like this–if you have no idea what the initial value should be, you can always assign it later.

Likewise, you can also modify the value of variables like this:

int first= 1;
System.out.println(first);
    
first = 2;
System.out.println(first);

In this case, first is initialized with a value of 1 but is then overwritten by the value of 2, so the output is

1
2

Section 3. Booleans

Expand

Booleans store boolean values (true or false). Booleans are the simplest primitive type because of this. However, they have very many use cases–they can be used as flags to check if something happened, or if a fact is valid or invalid.

boolean iPassedAllOfMyFinals = true;

Boolean operators

and ( && )

Sometimes we want to check if two conditions have been met. For example, you’ll want to leave the house if you have put your shoes on AND if you’ve brushed your teeth: 

boolean myShoesAreOn = true;
boolean brushedMyTeeth = true;
    
System.out.println(myShoesAreON && brushedMyTeeth); //Output: true

In this example, we are using the AND operator &&. We are checking if both of the boolean values are true, and if they both are true, then the statement evaluates to true.

However, if I didn’t put my shoes on yet, I would having something like this:

boolean myShoesAreOn = false;
boolean brushedMyTeeth = true;
    
System.out.println(myShoesAreOn && brushedMyTeeth); //Output: false

Since I didn’t put my shoes on yet, one of the boolean values are false, so the entire statement is false.

or ( | | )

What if we only care about one of the two conditions? For example, I want to check whether I went to the mall or I went to the park—as long as I went to either place, I know I had a busy day. 

boolean wentToPark = true;
boolean wentToMall = false;
    
System.out.println(wentToPark || went toMall); // Output: true

This is an example of logical or, | | .

In this case, I went to the park, but not the mall. The output is still true because at least one of the conditions was true. On the other hand, I could go to the mall and NOT the park
And it would still evaluate to true. If I didn’t go to either, then the output is false, and I likely didn’t go outside (at least in our made-up scenario).

not ( ! )

So far we’ve checked if some condition has been met, but sometimes we are interested if some condition has not been met. For example, I would like to know if I haven’t gone to college yet (I have), I could say: 

boolean wentToCollege = true;
System.out.println(!wentToCollege); //output: false

In this example, we are using logical not !
Since I went to college, and I am asking if I did NOT got to college, the output is false.

In other words, the not operator inverts the value of the boolean variable. If the value is currently true, then the output is false, and if the value is currently false, then the output is true.

combining it together

You will see that this can get pretty complex. Let’s say it’s final exam week, and in order to keep my scholarship I need the following conditions to be true:

  • I pass English AND Math
  • Alternatively, I can just pass Science.
  • I do not cheat

Science is weight mored because It’s worth 4 credit hours compared to the 2 credit hours each of English and Math. You could model something like this:

boolean passedEnglish = true;
boolean passedMath = false;
boolean passedScience = true;
boolean cheated = true;
    
System.out.println(((passedEnglish && passedMath) || passedScience) && !cheated);

You can kind of see how the conditions translate into boolean expressions. In short, since I cheated on the exam, the entire statement is false, and I lost my scholarship. Boohoo!

We will cover boolean expressions in more detail later.

Section 4. Chars

Expand

A char (short for character). it stores one single ASCII character. Emojis are not included in this range, so in order to print emojis you will have to use a String and escape a unicode character using \u.

char grade = 'F';

One interesting thing to note is that chars have a numerical value, so you can add integers to chars like the following:

char grade = 'a' + 1;
System.out.println(grade); //Output: b

The ASCII value of a is 97. When we add 1 to 'a', we the ASCII value turns into 98, which represents 'b'. So the output is now 'b'.

Section 5. Ints

Expand

An int is a 32-bit signed whole number.

int numSiblings = 2;

integers have a range of, roughly, ±231.

Quick question though–I said ints are stored as 4 bytes (32-bits). So why 231 and not 232? The answer is simple–one of the bits is reserved for the sign (+ or -). Thus the remaining 31 bits encode the value.

ints can be used in several mathematical operations. Here are the four basic arithmetic operations. 

int x = 5;
int y = 2;
    
System.out.println(x+y); // Output is: 7 (5 plus 2 is 7)
    
System.out.println(x-y); // Output is: 3 (5 minus 2 is 3)
    
System.out.println(x*y); // Output is: 10 (5 times 2 is 10)
    
System.out.println(x/y); // Output is: 2 (5 divided by 2 is... 2(???))

Makes sense right?–Wait a second, The division operation gave the wrong answer!

5/2 is equal to 2.5, so what gives? The answer is simple–integers do not work with decimal values, so they drop whatever is past the decimal point, so 2.5 ends up as 2.

There three other arithmetic operations that you likely haven’t used in math before.

Increment operator (++)

int x = 5;
x++; //x is now 6

The increment operator adds 1 to the current value. It’s a short hand way of writing

x = x + 1;

As Computer Scientists, we strive to write as little code as possible and perform as many things as possible.

Decrement operator (–)

int x = 5;
x--; //x is now 4

Just like the increment operator, the decrement operator subtracts 1 from the current value, and is short hand for

x = x - 1;

Modulo operator (%)

int x = 5;
int y = 2;
    
System.out.println(x%y); //output is 1

The modulo operator calculates the remainder. The example above shows: 5 divided by 2 has a remainder of 1.

This operator is super important in many areas of Computer Science–especially in Cryptography. With the modulo operator, you can check if any number is exactly divisible by another number (which can also reveal if the former is a prime number, since prime numbers can’t be divided evenly by anything besides 1 and themselves). 

int x = 450;
int y = 2;
    
System.out.println(x%y); //output is 0--we know x is even.

Section 6: Doubles

Expand

Doubles are numbers that have decimal places.

double circumference = 3.14159;
double weight = 191.2;
double angle = 0.011;

Doubles are a superset of integers–in other words, all integers are doubles (1 is 1.0, for example), but doubles extend beyond the range of values that integers can represent.

In fact, doubles have a range of ±1.798 × 10308, or if you wrote out the zeroes it would look like this: 

1,798,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000,000

With such an insane range of values, what’s the point of integers?

There are a couple reasons why you wouldn’t use a double over an int.

Memory

doubles are stored as 64-bit values, meaning they take up 4 bytes of space in memory–twice the size of integers. You can run into memory issues much faster with them.

Precision

I want you to guess the output of this code.

double a = 0.1;
double b = 0.2;
System.out.println(a + b);

Very clearly the answer is 0.3, right? No! The output of this code is 0.30000000000000004 because floating point numbers are also stored in binary.

Because 0.3 in binary is 0.01100110011…

The 0011 pattern repeats ad infinitum, but in reality we don’t have infinite bits to represent 0.3 so we end up with

0.011001100110011001100110011001100110011001100110011

Which is not exactly 0.3. This precision error is prevalent in so many languages that an entire website has documented it: https://0.30000000000000004.com/

Java has certain work arounds for this (like Double.round()), but in the end, if you should think very deeply about what data type represents your data the best, and what tradeoffs there are to selecting one over the other.

Section 7: Strings

Expand

I won’t be covering Strings in too much detail in this section, as this lesson is focused on primitive types. However, it’s important to note that you have been working with Strings this entire time!

String message = "Hello World"!;
System.out.println(message);

Instead of printing a new string in our println statement, we are creating a String variable, assigning it a value, then printing the value of the String. You can also do stuff like concatenation:

String a = "Foo";
String b = "Bar";
String c = a + b;

System.out.println(c); // Output is FooBar

In this example, b is concatenated to a, meaning it’s essentially combined into one String c.

Be careful: you can concatenate Strings with other data types, but they all turn into Strings.

int x = 5;
int y = 10;
String message = "Sum: " + x + y;
System.out.println(message);

Here we are adding x, and y to sum. However, the output is

Sum: 510

Here’s the rationale behind this: "Sum: " + 5 is concatenated first, and then the resulting String is concatenated with 10 (the value of y).

If we want to perform mathematical operations on x and y, we should surround them in parentheses:

int x = 5;
int y = 10;
String message = "Sum: " + (x + y);
System.out.println(message);

And now the output is

Sum: 15

Which is what we were expecting in the first place.
